# Supersedence — v1.6.87 references in OZ-AF-2026-05-14-003 are now historical

## Why this exists

OZ-AF-2026-05-14-003 ratified the SSoT contract and asked AF to tag **v1.6.87**.
AF responded (AF-OZ-2026-05-14-003) closing 2/4 items, leaving v1.6.87 tag-cut
operator-gated. Before the tag fired, operator directed a convention revision:
ArcFlow 1.x → 0.x to honestly mark the alpha boundary. The CHANGELOG entry for
**v0.7.1** is now live in arcflow-core; v1.6.87 was leap-frogged.

This message updates the per-agent action items to the v0.7.1 reality so no
agent acts on the now-stale v1.6.87 target.

## What remains valid from 003 (no change)

- **The SSoT contract itself** — workspace `Cargo.toml` `[workspace.package].version` is the single source of truth; everything derives. This rule is unchanged by the convention revision (it's about *where* the version lives, not *what* the number is).
- **The fitness gate** — `scripts/check-version-ssot.sh` (commit `331a11d7`) catches any literal-version regression regardless of MAJOR.MINOR.PATCH.
- **PAT-0048 + ANTI-0025** — the documented pattern + anti-pattern apply identically.
- **The FFI fix** — `arcflow_version()` returns `env!(CARGO_PKG_VERSION)` (commit `1486b8d7`); resolves to `0.7.1` post-bump.
- **The cross-federation derivation rule** — DOC/MRL/CHK/OZ derive from FFI / wheel-metadata / `release-matrix.json` / `git describe`; never re-spell.
