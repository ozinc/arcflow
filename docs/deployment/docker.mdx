# Deployment: Docker

Run ArcFlow in Docker — zero build tools, 30 seconds to first query.

The Docker image and `docker run` command come from the install matrix on
the [Installation page](/installation). Once you have the image name, the
patterns below show how to wire it up.

## Quick start

Substitute `$ARCFLOW_IMAGE` with the docker row from the install matrix.

```bash
docker run -p 7687:7687 -p 8080:8080 "$ARCFLOW_IMAGE"
```

## docker-compose

Set `image:` to whatever the install matrix shows for the docker row.

```yaml
services:
  arcflow:
    image: ${ARCFLOW_IMAGE}    # see /docs/installation
    ports:
      - "7687:7687"
      - "8080:8080"
    volumes:
      - arcflow-data:/data
    environment:
      - ARCFLOW_DATA_DIR=/data
      - ARCFLOW_LOG_LEVEL=info

volumes:
  arcflow-data:
```

```bash
docker compose up -d
```

## Connect from SDK

```typescript
import { open } from 'arcflow'

// In-process: embed ArcFlow directly (same machine, no Docker needed)
const db = open('./data/graph')
```

Or query the Docker container over HTTP:

```bash
curl -X POST http://localhost:8080/query \
  -d "MATCH (n:Person) RETURN n.name"
```

## Data persistence

Mount a volume to `/data` to persist graph data across container restarts:

```bash
docker run -v ./my-data:/data "$ARCFLOW_IMAGE"
```

## Image details

- **Base:** scratch (no OS, minimal attack surface)
- **Binary:** statically linked, <20MB
- **Architecture:** `linux/amd64`, `linux/arm64`

## Environment variables

| Variable | Default | Description |
|---|---|---|
| `ARCFLOW_DATA_DIR` | `/data` | Data directory |
| `ARCFLOW_LOG_LEVEL` | `info` | Log level (debug, info, warn, error) |
| `ARCFLOW_METAL_FORCE_UNAVAILABLE` | `false` | Disable Metal GPU |
