# Sessions & Transactions

## Session parameters

A session carries parameters that scope the statements run on it — most importantly the **acting principal and role** used for permission-based access control (PBAC):

```cypher
SET SESSION ACTOR 'svc-ingest' ROLE 'editor'
SET SESSION key = value           -- general session parameter (alias: SESSION SET key = value)
```

Reset session state with:

```cypher
SESSION RESET PARAMETERS        -- clear set parameters
SESSION RESET CHARACTERISTICS   -- reset transaction characteristics
SESSION RESET ALL               -- both
```

## Transaction control

ArcFlow supports ISO-GQL transaction statements:

```cypher
START TRANSACTION READ WRITE
  -- … statements …
COMMIT      -- or: ROLLBACK
```

- **`START TRANSACTION [READ ONLY | READ WRITE]`** — open an explicit transaction; declare the access mode.
- **`ISOLATION LEVEL SERIALIZABLE | REPEATABLE READ | READ COMMITTED | READ UNCOMMITTED`** — optional isolation clause (e.g. `START TRANSACTION READ WRITE ISOLATION LEVEL SERIALIZABLE`).
- **`COMMIT`** / **`ROLLBACK`** — finish the transaction.

Outside an explicit transaction, each statement commits on its own (autocommit).
