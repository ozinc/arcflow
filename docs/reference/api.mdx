# API Reference

Complete TypeScript SDK API. For Python, see [Bindings](/bindings) — same engine, plus the Python-only fast paths (`bulk_create_*`, `result.to_arrow / to_polars / to_pandas`, GIL-released threading).

## Module exports

```typescript
import { open, openInMemory, ArcflowError } from 'arcflow'
```

## `open(dataDir: string): ArcflowDB`

Open a persistent graph database. Data is WAL-journaled to disk.

```typescript
const db = open('./data/graph')
```

## `openInMemory(): ArcflowDB`

Open an in-memory graph database. Data is lost when the process exits. Ideal for testing.

```typescript
const db = openInMemory()
```

## `ArcflowDB`

The main database interface.

### `db.version(): string`

Returns the engine version string.

### `db.query(cypher: string, params?: QueryParams): QueryResult`

Execute a read query. Returns typed results. The result carries a `snapshotUri` field
identifying the snapshot the query observed — see [Snapshot-Pinned Reads](/concepts/snapshots).

```typescript
const result = db.query("MATCH (n:Person) RETURN n.name, n.age")
const result = db.query("MATCH (n {id: $id}) RETURN n", { id: 'p1' })
result.snapshotUri  // "arcflow://snapshot/9c3b…"
```

### `db.queryAt(snapshotUri: string, cypher: string, params?: QueryParams): TaggedQueryResult`

Replay a read query against a pinned historical snapshot. The result is bit-for-bit
identical to the original query against the same snapshot URI — full reproducibility.

```typescript
const a = db.query("MATCH (e:Entity {id: 'unit-01'}) RETURN e._confidence")
// ... mutations happen ...
const b = db.queryAt(a.snapshotUri, "MATCH (e:Entity {id: 'unit-01'}) RETURN e._confidence")
// b.rows is bit-for-bit identical to a.rows
```

Returns a `TaggedQueryResult { snapshotUri, rows, columns, ... }` — the same shape as
`QueryResult` with `snapshotUri` guaranteed to equal the URI passed in.

### `db.mutate(cypher: string, params?: QueryParams): MutationResult`

Execute a mutating query (CREATE, MERGE, SET, DELETE).

```typescript
db.mutate("CREATE (n:Person {name: $name})", { name: 'Alice' })
```

### `db.batchMutate(queries: string[]): number`

Execute multiple mutations under a single write lock. Returns count of mutations applied.

```typescript
const count = db.batchMutate([
  "MERGE (a:Person {id: 'p1', name: 'Alice'})",
  "MERGE (b:Person {id: 'p2', name: 'Bob'})",
])
```

### `db.isHealthy(): boolean`

Returns `true` if the database is operational. Returns `false` after `close()`.

### `db.stats(): GraphStats`

Returns node, relationship, and index counts.

```typescript
const s = db.stats()
console.log(s.nodes, s.relationships, s.indexes)
```

### `db.close(): void`

Close the database and flush WAL. All subsequent operations throw `ArcflowError`.

### `db.cursor(query: string, params?: QueryParams, pageSize?: number): QueryCursor`

Return a paginated cursor over a query. Useful for large result sets that shouldn't all be loaded into memory at once.

```typescript
const cursor = db.cursor("MATCH (n:Person) RETURN n.name, n.age", {}, 100)
while (!cursor.done) {
  const page = cursor.next()
  for (const row of page!.rows) { ... }
}
cursor.close()
```

### `db.subscribe(query: string, handler: SubscriptionHandler, options?: SubscribeOptions): LiveQuery`

Subscribe to a live delta stream for a query. The handler fires on every change with the added rows, removed rows, and full current result set. Uses a `CREATE LIVE VIEW` under the hood.

```typescript
const sub = db.subscribe(
  "MATCH (n:Alert) WHERE n.severity = 'critical' RETURN n.id, n.message",
  (event) => {
    console.log('new alerts:', event.added)
    console.log('resolved:', event.removed)
  }
)

// Stop when done
sub.cancel()
```

### `db.syncPending(): number`

Returns the number of mutations pending sync to the cloud. Returns 0 if sync is disabled or fully caught up.

### `db.fingerprint(): string`

Returns the SHA-256 hash of the current full graph state. Two graphs with identical content produce identical fingerprints regardless of insertion order. Useful for sync verification and integrity checking.

```typescript
const hash = db.fingerprint()
// "sha256:a1b2c3d4..."
```

### `db.ingestDelta(deltaJson: string): string`

Apply a structured delta (added nodes, removed nodes, added edges, removed edges) in a single atomic batch — bypasses the WorldCypher compiler entirely. Preferred for ingestion pipelines that track changes as deltas rather than full state. Sub-millisecond for typical batches.

Takes a JSON string with the delta payload; returns a JSON string with `DeltaStats` (`nodes_added`, `nodes_skipped_by_hash`, etc.). For typed args and results, use the [`CodeGraph`](/guides/code-intelligence) wrapper's `ingest()` method instead.

### `db.impactSubgraph(rootIdsJson: string, edgeKindsJson: string, maxDepth: number): string`

BFS blast-radius traversal — "what does changing this symbol break?" Bypasses the WorldCypher compiler. Takes JSON strings and returns a JSON string `{ nodes: [{ id, hop }] }`.

For the typed form, use `CodeGraph.impactSubgraph(rootIds: string[], edgeKinds: string[], maxDepth: number): ImpactSubgraph`:

```typescript
import { CodeGraph } from '@arcflow/sdk'
const cg = new CodeGraph(db)
const impact = cg.impactSubgraph(['fn_login'], ['CALLS', 'TESTED_BY'], 4)
for (const node of impact.nodes) {
  console.log(`hop ${node.hop}: ${node.id}`)
}
```

## `QueryResult`

```typescript
interface QueryResult {
  columns: string[]       // Column names
  columnTypes: ColumnType[]  // Parallel to columns: declared type of each column
  rows: TypedRow[]        // Typed row accessors
  rowCount: number        // Number of rows
  computeMs: number       // Execution time (ms)
  gqlstatus(): string     // ISO GQL status: "00000" = data, "02000" = no data
  snapshotUri: string     // arcflow://snapshot/<hex> — the snapshot this query observed
}

type ColumnType =
  // Scalars
  | 'unknown'                                            // Engine cannot statically type (mixed aggregates, mutating clauses)
  | 'null'
  | 'bool'
  | 'int'
  | 'float'
  | 'string'
  // Entity references — wrappers should NOT coerce these to int
  | 'nodeId'
  | 'relId'
  // Homogeneous lists — element type known
  | 'intList'
  | 'floatList'
  | 'stringList'
  // Spatial — six distinct tags, not a single "spatial" bucket
  | 'point'                                              // 2D / 2.5D point
  | 'point3d'
  | 'vector3d'
  | 'extent'                                             // axis-aligned bounding box
  | 'lineString'
  | 'polygon'
```

`columnTypes[i]` describes the value type of `columns[i]`. `'unknown'` is reported for columns the engine cannot statically type (mixed-type aggregates, mutating clauses, or expressions whose result type depends on runtime values). Bindings for languages with native list types use the `*List` tags to decide whether to parse a cell into a typed array versus leaving it as a string. The split spatial tags (`point` vs `point3d` vs `polygon` etc.) preserve geometry kind so wrappers don't have to re-detect it from the cell value.

`gqlstatus()` follows ISO/IEC 39075 GQL. `"00000"` means rows were returned; `"02000"` means the query succeeded but produced no rows (e.g. a `MATCH` with no matches, or a filter that eliminated everything).

`snapshotUri` is content-addressed and round-trippable. Pass it to `db.queryAt()` to
replay the same query against the same world state. See [Snapshot-Pinned Reads](/concepts/snapshots).

## `TaggedQueryResult`

Returned by `db.queryAt()`. Same shape as `QueryResult` — explicitly named for
provenanced replay paths where the snapshot URI is the contract.

```typescript
interface TaggedQueryResult extends QueryResult {
  snapshotUri: string     // Always equals the URI passed to db.queryAt()
}
```

## `MutationResult`

Extends `QueryResult` with mutation statistics.

```typescript
interface MutationResult extends QueryResult {
  nodesCreated: number
  nodesDeleted: number
  relationshipsCreated: number
  relationshipsDeleted: number
  propertiesSet: number
}
```

## `TypedRow`

### `row.get(column: string): string | number | boolean | null`

Get a typed value by column name. Supports both full (`n.name`) and short (`name`) column names.

### `row.toObject(): Record<string, string | number | boolean | null>`

Get all columns as a typed key-value object.

## `QueryCursor`

```typescript
interface QueryCursor {
  pageSize: number        // Rows per page
  pagesFetched: number    // How many pages retrieved so far
  done: boolean           // True when all pages exhausted
  next(): QueryResult | null   // Fetch next page (null if done)
  all(): QueryResult           // Collect all remaining pages into one result
  close(): void                // Release the cursor
}
```

## `LiveQuery`

```typescript
interface LiveQuery {
  viewName: string   // Internal live view name
  cancel(): void     // Stop the subscription and drop the live view
}
```

## `DeltaEvent`

```typescript
interface DeltaEvent {
  added: SubscriptionRow[]     // Rows newly entering the result set
  removed: SubscriptionRow[]   // Rows leaving the result set
  current: SubscriptionRow[]   // Full current result set
  frontier: number             // Monotonic mutation sequence number
}

type SubscriptionRow = Record<string, string | number | boolean | null>
type SubscriptionHandler = (event: DeltaEvent) => void
```

## `SubscribeOptions`

```typescript
interface SubscribeOptions {
  pollIntervalMs?: number   // How often to check for updates (default: 50ms, min: 10ms)
}
```

## `ArcflowError`

```typescript
class ArcflowError extends Error {
  code: string                // "EXPECTED_KEYWORD", "LOCK_POISONED", etc.
  category: ErrorCategory     // "parse" | "validation" | "execution" | "integration"
  suggestion?: string         // Recovery hint (when available)
}
```

### `ArcflowError.fromNapiError(err: unknown): ArcflowError`

Parse a raw napi error into a structured ArcflowError. Used internally by the SDK.
