# CREATE SKILL

Register a skill in the world model. Once registered, use [PROCESS NODE](/skills/process-node) to run it against matched nodes.

## Syntax

```cypher
CREATE SKILL <name>
  FROM PROMPT '<rule_or_instruction>'
  ALLOWED ON [<Label>, ...]
  TIER SYMBOLIC
```

## Example

```cypher
CREATE SKILL ZonePeerDetector
  FROM PROMPT 'Link two Robot nodes as ZONE_PEER if they occupy the same zone with combined confidence above 0.75'
  ALLOWED ON [Robot]
  TIER SYMBOLIC
```

## Parameters

| Parameter | Description |
|---|---|
| `<name>` | Unique skill name. Use versioned names (`MySkill_v2`) to track evolution. |
| `FROM PROMPT` | The rule or instruction the skill applies when evaluating node pairs. |
| `ALLOWED ON` | List of node labels this skill applies to. Scopes execution to relevant nodes. |
| `TIER` | Execution tier. `SYMBOLIC` is the active tier — pattern matching and rule-based logic. |

## Execution Tiers

| Tier | Status | Description |
|---|---|---|
| `SYMBOLIC` | Active | Pure logic, pattern matching, rule-based. Sub-millisecond. |
| `WASM` | Planned | Compiled logic in a sandboxed runtime. |
| `LLM` | Planned | Batched model inference for complex relationship judgements. |

## Notes

- The skill is stored in the graph immediately but does not execute on creation
- Registration and execution are separate — call `PROCESS NODE` to run it
- Use versioned names (`_v2`, `_v3`) when updating logic; old edges retain their `_skill_version`
- Multiple skills can be active for the same label simultaneously
