# Modeling a Social Graph

A social network is a world model of human connection — who knows whom, how many hops apart they are, and which nodes carry the most influence. This guide builds one from scratch: nodes with properties, directed relationships, multi-hop traversals, shortest path, and centrality scores.

Under 2 minutes. All in GQL.

## Create people

```cypher
CREATE (alice:Person {name: 'Alice', age: 30, city: 'London'})
CREATE (bob:Person {name: 'Bob', age: 25, city: 'Berlin'})
CREATE (carol:Person {name: 'Carol', age: 35, city: 'London'})
CREATE (dave:Person {name: 'Dave', age: 28, city: 'Paris'})
```

## Create relationships

Relationships are first-class objects — not foreign keys derived at query time, but stored edges with direction and properties.

```cypher
MATCH (a:Person {name: 'Alice'}), (b:Person {name: 'Bob'})
CREATE (a)-[:KNOWS]->(b)

MATCH (b:Person {name: 'Bob'}), (c:Person {name: 'Carol'})
CREATE (b)-[:KNOWS]->(c)

MATCH (a:Person {name: 'Alice'}), (d:Person {name: 'Dave'})
CREATE (a)-[:KNOWS]->(d)
```

## Who does Alice know directly?

```cypher
MATCH (a:Person {name: 'Alice'})-[:KNOWS]->(friend)
RETURN friend.name
```

## Friends of friends (2 hops)

```cypher
MATCH (a:Person {name: 'Alice'})-[:KNOWS*2..2]->(fof)
RETURN DISTINCT fof.name
```

The `*2..2` is a variable-length path — match exactly 2 hops. `*1..3` would match 1, 2, or 3 hops. No recursion needed, no self-join.

## Shortest path between two people

```cypher
MATCH p = shortestPath(
  (a:Person {name: 'Alice'})-[:KNOWS*]-(c:Person {name: 'Carol'})
)
RETURN length(p), [n IN nodes(p) | n.name] AS path
```

## Filter: people in London over 25

```cypher
MATCH (n:Person)
WHERE n.city = 'London' AND n.age > 25
RETURN n.name, n.age
```

## Aggregate statistics

```cypher
MATCH (n:Person)
RETURN count(n) AS total, avg(n.age) AS avg_age, collect(n.city) AS cities
```

## PageRank — who has the most influence?

```cypher
CALL algo.pageRank()
YIELD nodeId, score
MATCH (p:Person) WHERE id(p) = nodeId
RETURN p.name, score
ORDER BY score DESC
```

PageRank measures structural influence — which person, if removed, would most disconnect the network. In a larger graph this answers questions no property filter can.

## Optional: people Alice may know (OPTIONAL MATCH)

```cypher
MATCH (a:Person {name: 'Alice'})
OPTIONAL MATCH (a)-[:KNOWS*2..3]->(extended:Person)
WHERE NOT (a)-[:KNOWS]->(extended)
RETURN DISTINCT extended.name AS suggested_connection
```
