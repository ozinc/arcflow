# Agent Integration

ArcFlow speaks the language of AI agents -- structured errors with recovery suggestions, deterministic output (same query + same state = same results), and filesystem-native operation with no daemon process required. Coding agents (Claude Code, Codex, Gemini CLI) use the CLI binary — each command exits in <10ms, no session required. Cloud chat interfaces (ChatGPT, Claude.ai) use the MCP server. Every response is machine-parseable.

## CLI as control surface

Agents interact with ArcFlow through the CLI. Every query returns structured,
parseable output.

```bash
# Agent sends a query
echo "MATCH (n:Person) RETURN n.name, n.age" | arcflow
```

## Deterministic behavior

- Same query against same state = same results
- Replay of write sequence produces same graph
- No ambiguity in result schemas

## Structured errors

Every error is typed with machine-readable fields:

```json
{
  "class": "Validation",
  "code": "UNKNOWN_LABEL",
  "message": "Label 'Perso' not found",
  "failing_field": "label",
  "recovery_suggestion": "Check label spelling. Available: Person, Organization"
}
```

Agents parse `class` + `code` to decide recovery strategy without reading
the `message` field.

## EXPLAIN for planning

Before executing an expensive query, agents can inspect the plan:

```cypher
EXPLAIN MATCH (a)-[:KNOWS*1..5]->(b) RETURN b.name
```

## Snapshot/restore for checkpointing

Agents can checkpoint state before risky operations:

```
:snapshot ./before-migration.json
```

And roll back if something goes wrong:

```
:restore ./before-migration.json
```
