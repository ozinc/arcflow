# Get Started

The neural world model research community is trying to collapse simulation, memory, spatial grounding, and calibrated uncertainty into a single generative model. Papers have proposed baking persistence directly into the neural layer (MosaicMem), embedding 3D scene grounding inside the network (PERSIST), and adding uncertainty awareness to the generative system itself. This is the wrong architectural direction — these are not the same problem, and mixing them makes systems worse at both.

**ArcFlow is the correct architectural separation.** Simulation belongs in the neural layer — learned, generative, action-conditioned. Persistence, spatial indexing, temporal versioning, and confidence scoring belong in a separate, deterministic, queryable layer. The two layers have different latency requirements, different failure modes, different update rates. They should not be one thing.

Neural world models are extraordinary at simulating possible futures. They don't remember anything. **ArcFlow is what you build underneath them** — the persistence tier that stores what actually happened, spatially indexed, versioned to the sequence number, and queryable in microseconds.

```cypher
-- A query no neural world model can answer:
MATCH (s:Sensor)-[d:DETECTED]->(r:Robot)-[:OCCUPIES]->(z:Zone)
WHERE d.confidence >= 0.90 AND z.hazard > 0.15 AND r.battery < 30
RETURN r.name, z.name, avg(s.reliability * d.confidence) AS trust
```

This runs in ~0.1ms. Deterministic. Against exact history. The neural layer tells you what *might* happen next. ArcFlow tells you what *did* happen — which sensor, which detection, what confidence, which zone, at any moment in the past — with one query.

The AI infrastructure stack has three tiers. Foundation models handle language and perception. Neural world models simulate how the world evolves under actions. The operational world model is the persistence tier that has been missing — the layer where actual world state lives, versioned, spatially indexed, confidence-scored, queryable. That is ArcFlow.

It runs in-process — **no server, no network hop, no inference cluster**. Neural world models require GPU infrastructure and return in 100–1000ms. ArcFlow returns in microseconds, co-located with the agent, robot, or vehicle that needs it. For any real-time control loop, this is an architectural requirement, not a convenience.

Every mutation is versioned. Every fact carries `_confidence` and `_observation_class` (`observed`, `inferred`, `predicted`). Relationships are stored edges, not derived at query time. The full history is queryable with ISO GQL.

## Try instantly — zero install

Open **[ArcFlow Engine](https://oz.com/engine)** in your browser. Paste any GQL query — persistent storage, no install, no signup.

```cypher
CREATE (e:Entity {id: 'unit-01', x: 12.4, y: 8.7, _observation_class: 'observed', _confidence: 0.94})
CREATE (e:Entity {id: 'contact-x', x: 80.0, y: 90.0, _observation_class: 'predicted', _confidence: 0.38})
CALL algo.nearestNodes(point({x: 0, y: 0}), 'Entity', 5) YIELD node, distance RETURN node.id, distance
MATCH (e:Entity) WHERE e._confidence > 0.8 RETURN e.id, e.x, e.y
MATCH (e:Entity) AS OF seq 1 RETURN e.id, e._confidence
```

## Install

```bash
curl -fsSL https://oz.com/install | sh
```

Then:

```bash
arcflow --playground
```
