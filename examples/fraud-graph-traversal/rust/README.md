# Rust SDK · Fraud Graph Traversal

The same three pattern queries as the Python version (`../`) — fan-out,
fan-in, multi-hop chain — routed through the shipped Rust SDK surface.
Plus a fifth binary the Python recipe can't reach today: live fraud
alerts via `register_skill` and incremental mule clustering via
`LIVE CALL algo.connectedComponents`.

## When the Rust version pays off

- **Live fraud alerts during ingest.** `register_skill(..., NodeCreated
  {"Transfer"}, fan_in_alert_query)` fires the fan-in detection every
  time a new Transfer is created. The alert surfaces at mutation time;
  the polling fallback re-runs the query on a timer.
- **Incremental mule clustering.** `LIVE CALL algo.connectedComponents`
  runs connected-components as a standing graph algorithm — every new
  Transfer extends an existing component or unions two. The result set
  is current without recomputing the algorithm; reads from `MATCH (row)
  FROM VIEW __live_algo_connectedComponents` are zero-cost.
- **Provenance for compliance.** Every Transfer node, every alert that
  fired, every analyst note can carry `_source` + provenance edges that
  `arcflow_sdk::provenance_chain(&db, node_id)` walks in one call —
  audit response without a JOIN-chain reconstruction.

## Run

```bash
cargo run --release --bin 02-load
cargo run --release --bin 03-fan-detection
cargo run --release --bin 04-chain-detection
cargo run --release --bin 05-live-alerts
```

The Parquet fixture (`../data/transfers.parquet`) is generated by the
Python `00-make-sample.py` — run that once from the cookbook root first.

## Cargo layout

```
rust/
├── Cargo.toml                          # arcflow-sdk, arrow, parquet
├── src/
│   ├── lib.rs                          # Parquet → Account/Transfer schema
│   └── bin/
│       ├── 02-load.rs                  # bulk-ingest mirror
│       ├── 03-fan-detection.rs         # fan-out + fan-in
│       ├── 04-chain-detection.rs       # 4-hop layering pattern
│       └── 05-live-alerts.rs           # register_skill + connected-components
```
